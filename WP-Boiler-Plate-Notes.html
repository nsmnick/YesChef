WP Boiler Plate

Introduction

This boiler plate allows you to quickly create an empty Wordpress project with all the build scripts we use in place. The project is a blank empty canvas to work with. It contains all the standard plugins, functions usually required. There are directories of library code that is often used in projects that can be used if required, to make the development of sites quicker if required.

— Nick Morley - Oct 2019

Setup

  • Take a clone of the site from the Github repo. You have probably already done this, if you are reading this!
  • Create an empty database
  • Modify your wp-config.php to connect to your new database, you can change the wp-content and theme directory to match your project. You will need to change the settings in the top of the wp-config file accordingly. You can setup new database prefixes in your wp-config file if you would like.
  • When you first run the site, your standard empty database tables will be created for you.


Webpack

Webpack based compiler for SASS and JS that outputs to a Wordpress theme folder and can be cleanly removed from a development site for production.

  1. Note the webpack-builder folder in the site's root folder.
  2. In the terminal navigate to the webpack-builder folder.
  3. Run 'npm install'
  4. Edit the '.env' file and change the 'PUBLIC_PATH_ROOT' variable to the path of your Wordpress themes folder.
  5. Change 'PUBLIC_FOLDER_NAME' to your folder in your theme where the assets are to created.
  6. In the terminal run 'npm run build' to compile all assets for production or 'npm run watch' to run a development build that watches out for file changes. Live reload plugin is also supported.

There are a few things to take note of with this build process.

  • Webpack clears down the output assets folder completely on each build which means you do not have any redundent images or scripts in the public section of your site.
  • The Webpack main script that runs is webpack.config.js. You will see this takes assets/index.js as its inital source file to process. This links to the main scss file.
  • It extracts all the images referenced in the css includes them in the project. Any images used in the HTML directly have to be included in the the index.js file as an include for example "import favIcon from './images/favicon.png';"
  • The build script compresses images and provides a warning, if some images are larger than they should be.
